## Context (read first, ~3 min)

This repo is a vendored copy of `coleam00/adversarial-dev` — a three-agent GAN-style harness for long-running app development (Anthropic Harness Design pattern). The three roles are:

- **Planner** (`shared/prompts.ts::PLANNER_SYSTEM_PROMPT`) — product architect, writes spec.md from a 1-4 sentence user description
- **Generator** (`shared/prompts.ts::GENERATOR_SYSTEM_PROMPT`) — implements sprint features one at a time, commits per feature, self-evaluates
- **Evaluator** (`shared/prompts.ts::EVALUATOR_SYSTEM_PROMPT`) — runs the app, scores criteria 1-10, kills background processes, refuses generosity

These are agents in everything but documentation. Workbench is rewriting itself to make agents first-class entities with `agent.yaml` manifests per a 6-field contract:

1. **identity** — name, version, owner, purpose
2. **capability_surface** — allowed_tools, mcp_servers, max_context, max_turns
3. **memory_contract** — backend, namespace, retains, forgets
4. **eval_bindings** — must_pass, on_failure
5. **prompts** — system, on_tool_failure, on_max_context, style
6. **subagents** — declared dispatch graph (often `[]` for leaf agents)

Phase 1 here lays parallel documentation only. Existing `claude-harness/` and `codex-harness/` keep reading prompts from `shared/prompts.ts` — your new `agents/<name>/agent.yaml` files are documentation, NOT load-bearing yet. Workbench Phase 2 will make them load-bearing later.

```markdown
# agents/ — first-class agent manifests

> **New as of 2026-05-12 (Phase 1 of the agent-first rewrite).** This directory is the canonical home for agent.yaml manifests per the 6-field Agent Definition Contract (see https://greywiki.onlyarag.com/architecture/agents). Sister to the existing claude-harness/ and codex-harness/ directories which contain the Python/TypeScript harness code today.

## Why this exists

adversarial-dev today has three discrete agent roles (planner / generator / evaluator) defined as system prompts in shared/prompts.ts. The framework principle #10 ("agents are first-class") asks each role to be its own entity with a manifest, memory contract, and eval bindings. This directory is where that promotion lands.

See docs/agent-first-rewrite-plan.md for the full plan. For the workbench-side parent plan, see https://github.com/kombiz/workbench/blob/main/docs/agent-first-rewrite-plan-2026-05-12.md.

## Layout

- agents/README.md           — this file
- agents/planner/agent.yaml  — 6-field manifest for planner role
- agents/generator/agent.yaml — 6-field manifest for generator role
- agents/evaluator/agent.yaml — 6-field manifest for evaluator role

## Status

These manifests are draft 0.1.0 documentation-only. Existing claude-harness/ and codex-harness/ continue to read prompts from shared/prompts.ts. Phase 2 of the workbench rewrite will produce a manifest loader that makes these files load-bearing.

NEEDS-DECISION markers in the manifests are valid in drafts. They block promotion to 1.0.0 but not commit.
```

### File 3: `agents/planner/agent.yaml`

```yaml
identity:
  name: adversarial-planner
  version: 0.1.0
  owner: kombiz
  purpose: |
    Expand a 1-4 sentence user description into a full product spec
    organized into sprints. Stay high-level: feature names + user
    stories, not implementation details. Output is spec.md.

capability_surface:
  allowed_tools: [Write]
  mcp_servers: []
  max_context: 16000
  max_turns: 1                    # planner is one-shot

memory_contract:
  backend: filesystem
  namespace: workspace/${workDir}
  retains: [spec.md]
  forgets: []

eval_bindings:
  must_pass: [spec-shape, sprint-count-bounded, no-implementation-leak]
  on_failure: report

prompts:
  system: shared/prompts.ts::PLANNER_SYSTEM_PROMPT
  on_tool_failure: NEEDS-DECISION
  on_max_context: NEEDS-DECISION
  style: markdown-spec

subagents: []
```

### File 4: `agents/generator/agent.yaml`

```yaml
identity:
  name: adversarial-generator
  version: 0.1.0
  owner: kombiz

    Implement sprint features one at a time per the active contract.

  system: shared/prompts.ts::GENERATOR_SYSTEM_PROMPT
```

### File 5: `agents/evaluator/agent.yaml`

```yaml
identity:
  name: adversarial-evaluator
  version: 0.1.0
  owner: kombiz
  purpose: |
    Run the app, probe for failures, score each contract criterion 1-10
    on the threshold (default 7). Refuse generosity. Output structured
    EvalResult JSON. Kill background processes before returning.

capability_surface:
  allowed_tools: [Read, Bash, Glob, Grep]   # NO Write — evaluator must not edit the app
  mcp_servers: []
  max_context: 100000
  max_turns: 50

memory_contract:
  backend: filesystem
  namespace: workspace/${workDir}
  retains: [eval-result.json, feedback.md]
  forgets: [background-processes]   # mandatory cleanup of spawned servers

eval_bindings:
  must_pass: [meta-eval/honest-scoring, meta-eval/no-write-tool-use]
  on_failure: harness-marks-sprint-failed

prompts:
  system: shared/prompts.ts::EVALUATOR_SYSTEM_PROMPT
  on_tool_failure: report-as-criterion-failure
  on_max_context: NEEDS-DECISION
  style: json-only

subagents: []
```
